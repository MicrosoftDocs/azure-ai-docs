This article explains how data is processed when you use Claude models in Microsoft Foundry. Data handling depends on the hosting option you select when deploying the model:

- **Hosted on Azure**
- **Hosted on Anthropic infrastructure**

Claude models in Microsoft Foundry are third-party Marketplace offerings from Anthropic. Anthropic is the seller, operator, and data processor for Claude prompts and outputs. Microsoft provides the Foundry experience, Azure infrastructure, Marketplace transactions, and billing services. When you transact for Claude model in Microsoft Foundry, you agree to Anthropic's terms of use.

## How data processing differs by hosting option

### Hosted on Azure

For Claude models **Hosted on Azure**, Azure infrastructure processes prompts and outputs, including request ingress, API services, and GPU inference.

The selected Azure geography stores data at rest. For supported Data Zone Standard offers, processing is scoped to the applicable data zone.

Automatic safeguards flag content that might be sent to Anthropic Trust & Safety for review. Anthropic personnel might review customer content on an exception basis to investigate potential safety violations, subject to Anthropic's applicable terms.

### Hosted on Anthropic infrastructure

For Claude models **Hosted on Anthropic infrastructure**, Anthropic-managed infrastructure processes prompts and outputs.

Data might be processed outside of Azure, including outside of your selected Azure region for operational purposes such as performance, reliability, and capacity management. To learn about the terms that govern data processing, see [Anthropic's Data processing Addendum](https://www.anthropic.com/legal/data-processing-addendum) and [Anthropic's Commercial Terms of Service](https://aka.ms/anthropic_tandc).

## What data does Anthropic process for Claude models in Microsoft Foundry?

Anthropic is the data processor for prompts and outputs for Claude models in Microsoft Foundry.

For Claude models **hosted on Azure**, Azure infrastructure processes prompts and outputs. Anthropic doesn't process prompts and outputs on Anthropic-managed infrastructure, except when automatic safeguards flag content and send it to Anthropic Trust & Safety for review.

For Claude models **hosted on Anthropic infrastructure**, Anthropic-managed infrastructure processes prompts and outputs. Your use of the model (along with Anthropic's accountability for the model and its outputs) is subject to the terms of use Anthropic provides for the model. Anthropic acts as the data processor for prompts and outputs sent to, and generated by, an Anthropic Claude API.

## What data does Microsoft process for Claude models in Microsoft Foundry?

Microsoft provides the Foundry deployment experience, API endpoint integration, Azure infrastructure where applicable, Marketplace transaction flow, metering, billing, and related service operations.

Microsoft also collects billing, usage, customer contact, and transaction information for Marketplace operations. Microsoft may share customer contact information, transaction details, and usage volume with Anthropic so that Anthropic can operate, support, and communicate with customers about the model.

Microsoft processes data for these services under the [Microsoft Products and Services Data Protection Addendum (DPA)](https://www.microsoft.com/licensing/docs/view/Microsoft-Products-and-Services-Data-Protection-Addendum-DPA) and applicable Marketplace terms. For deployments of Claude models in Microsoft Foundry hosted on Azure, Microsoft operates as a subprocessor subject to Anthropic.

## Where can I learn about harmful content screening?

Claude models in Microsoft Foundry use Anthropic safety systems and safeguards. To learn more about harmful content screening, safety review, and Anthropic-specific processing, see Anthropic's documentation and the Anthropic terms presented during deployment.
