# Azure Document Intelligence known issues and troubleshooting

Azure Document Intelligence is updated continuously, and service changes can affect feature behavior and capability limits. This article tracks known issues in Azure Document Intelligence, describes their impact, and provides mitigation or resolution guidance. Before you submit a support request, review the list to determine whether your issue is a known condition and to identify the recommended remediation.

## Service-level outages and notifications

Azure status and Azure Service Health publish service-level outages and regional disruptions. Use these services to validate current impact, track incident lifecycle updates, and configure proactive notifications for your subscriptions and resources.

* **Service-level outages**: [Azure status page](https://azure.status.microsoft/status).
* **Outage notifications and alerts**: [Azure Service Health Portal](https://azure.microsoft.com/status/).

## Current known issues

The following table lists active, service-confirmed issues in Azure Document Intelligence. Each entry includes the issue category, observed behavior, documented workaround, and initial publish date.

| Issue ID | Category | Title | Description | Workaround | Publication date |
| --- | --- | --- | --- | --- | --- |
| **2006** | Model accuracy | **Table recognition fails when a single page contains tables in different orientations** | The current pipeline generates a single unified rotation angle per image or page and applies it to all detected tables. Multiple table orientations within the same page aren't supported. Because layout is the first stage in the recognition pipeline, custom models don't mitigate this limitation. **Models affected:** All models (layout-stage limitation). | Preprocess the document before submission: detect the dominant orientation of each table region (for example, by inspecting PDF content-stream rotation values or applying image-analysis rotation detection), split the page into separate images—one per distinct orientation—and submit each as an independent Analyze request. Merge the structured results in your application layer. | June 2026 |
| **2005** | Model accuracy | **Table extraction missing rows for single-row tables** | Single-row constructs aren't classified as tables in the current design. Rows may be missing from extraction output when the source document defines a table as a single row. This behavior is by design and may be refined in future releases. **Models affected:** All models.| Single-row table content is surfaced in the `paragraphs` or `keyValuePairs` arrays of the Analyze response rather than the `tables` array. Consume those arrays and retrieve the content. Alternatively, use the `boundingRegions` polygon coordinates from the `words` or `lines` arrays to reconstruct the row structure programmatically in post-processing. | June 2026 |
| **2004** | Model accuracy | **OCR character ambiguity: 1 vs l and O vs 0** | Visually ambiguous character pairs (`1`/`l` and `O`/`0`) can be misclassified, particularly in serial-number-like fields that lack semantic context. The model applies linguistic context to improve accuracy for natural-language strings (for example, correcting `6OO` to `600` or `App1e` to `Apple`), but this heuristic isn't effective for alphanumeric identifiers. **Models affected:** All models. | Apply field-specific post-processing validation on the extracted `content` value. For serial-number or identifier fields, use a regular expression or character-allowlist check against the expected character set and apply substitution rules where `1`/`l` or `O`/`0` confusion is domain-predictable. Additionally, inspect per-word `confidence` scores in the `words` array to identify low-confidence spans and prioritize for targeted human review or automated correction. | June 2026 |
| **2003** | Model accuracy | **OCR unable to detect superscript formatting** | Superscript-formatted characters aren't recognized as superscript. They're extracted as normal inline text without positional or formatting differentiation. **Models affected:** All models. | No model-level workaround is available; superscript detection is an OCR-stage limitation that custom model training doesn't address. As a post-processing alternative, compare the `polygon` or `boundingBox` coordinates of adjacent `word` spans within a `line` object in the Analyze response. Words whose vertical baseline is significantly elevated relative to surrounding words in the same line can be identified as likely superscripts and flagged or annotated accordingly in your application. | June 2026 |
| **2002** | Model accuracy | **Dash (-) character misrecognized or returned as "1"** | An intermittent model quality issue causes the dash (`-`) character to be misrecognized or returned as the digit `1`. **Models affected:** All models. | No model-level workaround is available; this issue is an OCR-stage misrecognition that custom model training doesn't resolve. Apply post-processing validation on fields where a dash is a valid or expected value: use a regular expression or positional heuristic to identify isolated `1` characters that appear in separator or null-value positions (for example, between alphanumeric tokens or in fixed-width structured fields), and substitute them with a dash. Use per-word `confidence` scores to prioritize which extractions require correction. | June 2026 |
| **2001** | Model accuracy | **Nested table structure not supported** | Document Intelligence prebuilt layout models don't support nested tables. Only the outermost table structure is extracted; inner nested tables aren't recognized as discrete table objects. **Models affected:** All models. | Custom extraction models and Content Understanding analyzers use the layout model as their first processing stage, so custom training doesn't resolve nested table detection. As a post-processing alternative, use the `boundingRegion` polygon coordinates returned for each detected `table` and `cell` object in the Analyze response to identify cell regions that spatially contain another detected table's bounding polygon. Use this overlap to reconstruct the parent-child nesting relationship programmatically in your application. | June 2026 |
| **1005** | Model training | **Model training stuck in *Not started* status** | If a model training job fails before execution, the failed job may remain visible in the model list with status **Not started**. This state results from backend lifecycle handling, not customer configuration. | The backend automatically clears failed training jobs after approximately seven (7) days.<br><br>If earlier removal is required, submit an [Azure portal support request](https://ms.portal.azure.com/?quickstart=true#view/Microsoft_Azure_Support/HelpAndSupportBlade/~/overview). CSS or PG engineering can execute backend deletion of the orphaned training record. | April 2026 |
| **1004** | Model training | **HTTP 500 – InternalServerError during model training (Neural or Template, Preview/GA APIs)** | Model training can fail with **HTTP 500 / InternalServerError**. This failure typically occurs when managed identity isn't configured correctly on the resource, especially in newer training flows. | Enable **Managed Identity** on your Document Intelligence resource.<br><br>Verify that the identity has the required permissions.<br><br>Follow the official [managed-identity configuration guidance](../authentication/managed-identities.md). | April 2026 |
| **1003** | Model management | **Model Exists error when copying a model** | Copying a custom model can fail with a **Model Exists** error. This error occurs when a broken or partially created model remains on the target resource. Customers can't resolve this condition themselves. | No customer-side workaround is available.<br><br>Submit a [support request](mailto:formrecog_contact@microsoft.com) so the product group can remove the broken model from the backend. | April 2026 |
| **1002** | Encryption | **ServiceUnavailable error when using CMK** | Operations that rely on **Customer‑Managed Keys (CMK)** can intermittently fail with a **service unavailable** error. This failure occurs when the storage service uses stale cached identity or key material. | **Option A:** Refresh the `CMK` cache. Disable and re-enable `CMK`, assign a new key, or associate a new managed identity.<br><br>**Option B:** Switch to **Microsoft-Managed Keys (MMK)**: Navigate to your Document Intelligence resource in the [Azure portal](https://portal.azure.com/). **Resource Management** → **Encryption** → **Microsoft Managed Keys**. | April 2026 |
| **1001** | File types | **Content isn't supported error with Excel files** | Excel files are documented as a supported input type. Requests can fail with **Content isn't supported** when file content exceeds an internal character limit. This condition usually occurs with large or densely populated spreadsheets. | **Excel files larger than *~8,000,000 characters* are rejected**.<br><br>**Option A:** Split large Excel files into smaller files and submit them separately.<br><br>**Option B:** Switch to Azure [Content Understanding](../../content-understanding/overview.md), which uses improved parser logic for these files. | April 2026 |
